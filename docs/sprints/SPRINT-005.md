# Sprint 005: IPC Input Validation

## Sprint Theme

**Trust the renderer less. Fail earlier in main.**

---

## Overview

TypeScript types are erased at runtime. An IPC handler declared as `(_event, id: string)` receives
`unknown` at runtime — the renderer could pass `null`, an empty string, a 100 MB payload, or an
invalid enum value. Today, most handlers in `ipc/index.ts` and `ipc/settingsHandlers.ts` pass those
values straight to the DB or subprocess layer without checking them.

This sprint adds a lightweight `validate.ts` utility in `src/main/ipc/` (no new npm dependencies —
Node built-ins only) and applies it to every handler that accepts user-controlled input. Invalid
inputs throw a descriptive `Error`; Electron surfaces thrown errors as rejected promises to the
renderer. The renderer side is not touched — the main process is the trust boundary.

Sprint 004 (At-Rest Database Encryption) is now complete. The `settings:encryption:*` handlers it
added are out of scope for this sprint — they were written with inline validation in Sprint 004. This
sprint covers only the pre-existing handlers in `index.ts` and `settingsHandlers.ts`.

---

## ID Class Taxonomy

Not all IDs in the codebase are UUIDs. Before applying validation, the two categories must be clear:

| Category | Examples | Validator |
|---|---|---|
| **Entity IDs** — generated by `generateId()` (`randomUUID()`) in production; all integration test fixture IDs should be valid UUIDs | session IDs, composition IDs, voice IDs, message IDs, custom provider IDs, custom tone IDs, template IDs | `requireId` (UUID regex) |
| **Symbolic keys** — human-readable constants or provider strings, not generated by `generateId()` | built-in tone preset keys (`'collaborative'`), provider strings (`'anthropic'`), `voiceType` values | `requireNonEmptyString` or `requireEnum` |

`defaultTone` and `toneOverride` can be either category (preset key or custom UUID) — validate
as `requireNonEmptyString` with a length cap; do not apply UUID regex.

---

## Use Cases

1. **Malformed ID**: renderer passes `""` or `null` as a composition ID → handler throws `"Invalid compositionId: must be a valid UUID"` before touching the DB
2. **Invalid enum**: renderer passes `"roundrobin"` as a session mode → handler throws `"Invalid mode: must be one of: conductor, broadcast"`
3. **String overflow**: renderer passes a 10 MB string as a session name → handler throws `"name exceeds maximum length of 200 characters"`
4. **CLI injection attempt**: renderer passes `"../../malicious"` to `settings:testCliVoice` → handler throws `"Invalid command: must contain only alphanumeric characters, dots, hyphens, or underscores"` before `spawnSync` is called
5. **Non-boolean archived flag**: renderer passes `1` (integer) as the `archived` flag → coerced to `true` via `coerceBoolean`
6. **Invalid base URL**: renderer passes `"ftp://evil"` as a custom provider URL → handler throws `"Invalid baseUrl: must be a valid http or https URL"`

## Architecture

### `src/main/ipc/validate.ts` — the validation utility

```
Constants (exported):
  UUID_RE = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i
  CLI_COMMAND_RE = /^[a-zA-Z0-9][a-zA-Z0-9._-]*$/

Length constants (exported):
  MAX_NAME         = 200   — session, composition, custom provider name
  MAX_SHORT_NAME   = 100   — tone name, template name, display name, pronouns
  MAX_PROVIDER     = 50    — provider string, slug
  MAX_MODEL        = 200   — model identifier string
  MAX_CLI_COMMAND  = 100   — CLI command name
  MAX_DESCRIPTION  = 2000  — tone description, conductor context prose
  MAX_CONTENT      = 50000 — system prompt template content
  MAX_CONDUCTOR_CONTEXT = 10000 — UserProfile.conductorContext
  MAX_AVATAR       = 500000 — base64 avatar data URL
  MAX_URL          = 2000  — custom provider base URL
  MAX_MESSAGE_CONTENT = 100000 — message content

Helpers (exported):
  requireId(value: unknown, name: string): string
    → typeof value === 'string' && UUID_RE.test(value) → return value
    → else throw Error(`Invalid ${name}: must be a valid UUID`)

  requireString(value: unknown, name: string, maxLength: number): string
    → typeof value !== 'string' → throw Error(`Invalid ${name}: must be a string`)
    → value.length > maxLength → throw Error(`${name} exceeds maximum length of ${maxLength}`)
    → else return value

  requireNonEmptyString(value: unknown, name: string, maxLength: number): string
    → delegates to requireString; additionally throws Error(`${name} is required`) if value.trim() === ''

  requireEnum<T extends string>(value: unknown, name: string, allowed: readonly T[]): T
    → typeof value === 'string' && allowed.includes(value as T) → return value as T
    → else throw Error(`Invalid ${name}: must be one of: ${allowed.join(', ')}`)

  coerceBoolean(value: unknown, name: string): boolean
    → boolean → return value
    → 1 / '1' / 'true' → return true
    → 0 / '0' / 'false' / undefined / null → return false  ← undefined/null produce false because
      these args are optional list/archive filters that default to false in the current IPC contract
    → else throw Error(`Invalid ${name}: must be a boolean`)

  requireInteger(value: unknown, name: string, min: number, max: number): number
    → Number.isInteger(value) && value >= min && value <= max → return value
    → else throw Error(`Invalid ${name}: must be an integer between ${min} and ${max}`)

  requireCliCommand(value: unknown, name: string): string
    → delegates to requireNonEmptyString(value, name, MAX_CLI_COMMAND)
    → additionally: if !CLI_COMMAND_RE.test(value) → throw Error(`Invalid ${name}: must contain only alphanumeric characters, dots, hyphens, or underscores`)

  requireUrl(value: unknown, name: string, allowedProtocols: string[]): string
    → typeof value !== 'string' → throw Error(`Invalid ${name}: must be a string`)
    → try new URL(value) → if protocol not in allowedProtocols → throw Error(`Invalid ${name}: must be a valid ${allowedProtocols.join(' or ')} URL`)
    → URL parse throws → throw Error(`Invalid ${name}: must be a valid ${allowedProtocols.join(' or ')} URL`)
    → else return value

  requireArray(value: unknown, name: string): unknown[]
    → Array.isArray(value) → return value
    → else throw Error(`Invalid ${name}: must be an array`)

  requireObject(value: unknown, name: string): Record<string, unknown>
    → typeof value === 'object' && value !== null && !Array.isArray(value) → return value as Record<string, unknown>
    → else throw Error(`Invalid ${name}: must be an object`)

Composite validators (exported):
  requireMessageShape(value: unknown): Message
    → requireObject(value, 'message')
    → requireId(value.id, 'message.id')
    → requireId(value.sessionId, 'message.sessionId')
    → requireEnum(value.role, 'message.role', ['conductor', 'voice', 'system'])
    → requireNonEmptyString(value.content, 'message.content', MAX_MESSAGE_CONTENT)
    → requireInteger(value.roundIndex, 'message.roundIndex', -1, 100000)
    → requireInteger(value.timestamp, 'message.timestamp', 0, Number.MAX_SAFE_INTEGER)
    → value.voiceId: null | string — if not null, requireNonEmptyString
    → value.voiceName: null | string — if not null, requireNonEmptyString

  requireCompositionVoiceShape(value: unknown, index: number): CompositionVoice
    → requireObject(value, `voices[${index}]`)
    → requireNonEmptyString(value.provider, `voices[${index}].provider`, MAX_PROVIDER)
    → requireNonEmptyString(value.displayName, `voices[${index}].displayName`, MAX_SHORT_NAME)
    → requireInteger(value.order, `voices[${index}].order`, 0, 1000)
    → requireString(value.color, `voices[${index}].color`, 30)
    → requireString(value.avatarIcon, `voices[${index}].avatarIcon`, MAX_SHORT_NAME)
    → if value.model present: requireNonEmptyString
    → if value.systemPrompt present: requireString(value, name, MAX_CONTENT)
    → if value.cliCommand present: requireCliCommand
    → if value.cliArgs present: requireArray; each element requireNonEmptyString(, MAX_CLI_COMMAND)
    → if value.customProviderId present: requireId
    → if value.systemPromptTemplateId present: requireId
    → if value.toneOverride present: requireNonEmptyString(, MAX_SHORT_NAME)

  requireCompositionData(value: unknown): Omit<Composition, 'id' | 'createdAt' | 'updatedAt' | 'archived'>
    → requireObject(value, 'data')
    → requireNonEmptyString(value.name, 'name', MAX_NAME)
    → requireEnum(value.mode, 'mode', ['conductor', 'broadcast'])
    → requireEnum(value.continuationPolicy, 'continuationPolicy', ['none', 'prompt', 'auto'])
    → requireInteger(value.continuationMaxRounds, 'continuationMaxRounds', 1, 3)
    → requireArray(value.voices, 'voices') + each element via requireCompositionVoiceShape

  requirePartialCompositionData(value: unknown): Partial<Pick<Composition, 'name' | 'mode' | 'continuationPolicy' | 'continuationMaxRounds'>>
    → requireObject(value, 'data')
    → validates only fields present in value (same rules as requireCompositionData for each)

  requireUserProfileShape(value: unknown): Omit<UserProfile, 'updatedAt'>
    → requireObject(value, 'profile')
    → requireString(value.conductorName, 'conductorName', MAX_NAME)
    → requireString(value.pronouns, 'pronouns', MAX_SHORT_NAME)
    → requireString(value.conductorContext, 'conductorContext', MAX_CONDUCTOR_CONTEXT)
    → requireNonEmptyString(value.defaultTone, 'defaultTone', MAX_SHORT_NAME)
    → requireString(value.conductorColor, 'conductorColor', 30)
    → requireString(value.conductorAvatar, 'conductorAvatar', MAX_AVATAR)
```

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Short fixture IDs in integration tests cause cascading changes across unrelated tests | Medium | Medium | Named UUID constants defined once per file; `sessionHandlers.integration.test.ts` is also in scope and was not listed in the original drafts |
| Over-validation breaks legitimate inputs | Low | Medium | Length caps are generous; only structural checks added; edge cases covered in DoD |
| `requireCompositionData` rejects a valid Composition shape | Low | Medium | Unit test `requireCompositionData` with the same fixture shapes used in integration tests before wiring into handler |
| Sprint 004 encryption handlers accidentally touched | Low | High | Explicitly scope to pre-existing handlers; `settings:encryption:*` block in `settingsHandlers.ts` is out of bounds |
| `continuationMaxRounds` validation conflicts with future UI changes that allow > 3 | Low | Low | Constant is `CONTINUATION_MAX_ROUNDS_LIMIT = 3` in `shared/constants.ts`; validator should use the constant, not a hard-coded `3` |

## Security Considerations

- **`SETTINGS_TEST_CLI_VOICE`** is the highest-risk handler — it calls `spawnSync`. Even though `spawnSync(command, args)` does not invoke a shell, unchecked command strings could name unintended binaries or contain path traversal sequences. `requireCliCommand` with `CLI_COMMAND_RE` prevents path separators, spaces, and shell metacharacters — it does not prevent execution of any binary on `PATH` by name, which is an accepted constraint for a CLI voice probing feature.
- **ID injection**: UUID format validation ensures no SQL-significant characters appear in IDs — even though parameterized queries already prevent injection, this closes the class entirely.
- **String overflow**: capping strings before DB writes prevents degenerate inputs from exhausting disk or memory.
- **URL protocol restriction**: limiting `baseUrl` to `http:`/`https:` prevents `file://` and `javascript:` protocol abuse via custom provider fetch.
- **Error messages**: errors describe the invalid field without echoing secrets or untrusted payload content.
- **Enum validation prevents DB CHECK constraint violations**: caught at the IPC layer with a cleaner error message than a raw SQLite constraint failure.

| Question | Decision |
|---|---|
| `continuationMaxRounds` range | `1..3`, using `CONTINUATION_MAX_ROUNDS_LIMIT` constant |
| Boolean coercion policy | `coerceBoolean` for all boolean IPC args |
| `saveProviderConfig.provider` restriction | Non-empty string, 50-char cap only — no enum restriction |
| `requireCliCommand` regex | `/^[a-zA-Z0-9][a-zA-Z0-9._-]*$/` |
| `UPDATE_DISMISS` validation | Leave existing `VERSION_PATTERN` gate unchanged |
| `baseUrl` validation | `requireUrl` with `['http:', 'https:']` protocol allowlist |
